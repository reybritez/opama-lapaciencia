Este proyecto incluye:


Frontend JavaScript build pipeline, utiliza Webpack para empaquetar el archivo JavaScript ./asset/JavaScript/site.js Webpack está configurado para traducir; .js | .jsx y empaquetarlos en un formato legible por el navegador en ./static/JavaScript/site-bundle.js

RestFul API, que utiliza DRF, esto le ayuda a obtener su API en funcionamiento rápido. Un ejemplo de creación de una API a partir de un archivo excel ./applications/api/views.py

MarkDown/HTML .md Content Management System, Esta aplicación sirve archivos .md a tus plantillas Django, a través de ./applications/content/views.py esto se utiliza para crear artículos y contenido para el proyecto más fácilmente. Este es un método muy simplista para CMS pero por eso es bueno, porque tienes control total sobre el contenido y la entrega. Hay muchos aspectos negativos, como tener que volver a comprometer su aplicación en la producción, pero le sugerimos que se concentre en la calidad de su contenido SEO en lugar de la cantidad.

Django: SEO & Sitemap que utiliza Django para crear una página /sitemap.xml, que se puede registrar con los motores de búsqueda como GOOG, para añadir urls a su http://127.0.0.1:8000/sitemap.xml. Visita ./applications/website/SiteMap.py Los meta tags pueden ser anulados en las plantillas, por ejemplo ./templates/applications/content/content_django.html Tl Los meta tags html por defecto deben ser escritos en los 'bloques django' en la página ./templates/base.html esto sirve como la página html base para el proyecto.

Website Django Application sta aplicación es para cualquier página web genérica; términos, privacidad, acerca de, etc. Las plantillas de sitios web son básicamente para páginas estáticas / páginas renderizadas backend.

La idea con esto es que nos permite construir una aplicación rápidamente ya que parte de la infraestructura vital, ya se encuentra establecida de una manera bien estructurada siguiendo las buenas prácticas django y web. Usted puede integrar fácilmente react en cualquiera de sus plantillas django, teniendo la opción de utilizar JavaScript moderno o no.
